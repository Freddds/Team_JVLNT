Team JVLNT - Software Setup and User Guide

Description of this guide

This guide explains how to set up the required development environment for the MSE project. It includes installation steps for Visual Studio Code, PlatformIO, and Git, as well as the recommended configuration for working with the Pololu 3pi+ 32U4 platform.

Follow the steps carefully to ensure all team members use a consistent setup, so the project builds and runs correctly on all machines.

Yellow → Code snippets / terminal commands

Blue → Links / external resources

Red → Warnings / critical notes

1.1 Install Required Software

  • Install Git
  • Install VS Code
  • Install PlatformIO extension in VS Code: Installation Guide (only step 1 and 2)

1.2 Git Installation Settings

  • Use VS Code Studio as Git's default editor
  • Let Git decide default branch naming
  • Git from the command line and also from 3rd-party software
  • Use bundled OpenSSH
  • Use the native Windows Secure Channel library
  • Checkout Windows Style (CRLF handling)
  • Use MinTTY terminal
  • Rebase as default pull strategy
  • Git Credential Manager
  • Enable File System Caching

2. Verify Git Installation

Open terminal and run:

git --version

If a version appears → OK ✔

7. Workflow (Team Structure)

The project uses a staged branch workflow to keep the main robot code stable while allowing collaborative development on a separate integration branch. All development changes are collected in the testing branch before being reviewed and merged into main.

Repository Structure

  • main: Stable, tested robot code used for releases
  • testing: Integration branch where all development changes are collected and tested

Development Workflow

git pull origin testing
↳
git checkout testing
↳
work/code
↳
git add .
↳
git commit -m "descriptive message"
↳
git push origin testing

Pull Requests (testing → main)

Pushing to the testing branch does NOT automatically create a Pull Request. A Pull Request must be created manually on GitHub.

  1. Push changes to the testing branch using the commands above.
  2. Open the repository on GitHub in a browser.
  3. Set base to main and compare to testing.
  4. Create the Pull Request.
  5. Main editors review the changes and merge them into main.

Main Editors Workflow

Main editors are responsible for reviewing and merging tested changes from testing into main. 

git checkout main
git merge testing
git push origin main

8. Common Issues

  • “git not found” → Git not installed properly
  • “pio not found” → PlatformIO extension missing
  • push rejected → run git pull first
